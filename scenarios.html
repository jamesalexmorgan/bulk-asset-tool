Scenario 1

Load all standard page assets under a specific node into the BAT and view the attributes/metadata

  • Go to the Load Assets tab in the control panel
  • Enter your root node id into the Asset Ids textarea
  • Leave Limit types checked (so that it doesn't load all assets like content divs)
  • Select Standard Page from the Limited asset type codes select box (this list can be configured in the Config tab)
  • Leave Include roots unchecked so that it doesn't load the parent assets
  • Click Load assets

This will load all the attributes and metadata of the assets into the BAT

  • In the Display fields select box, select a few attributes and metadata fields

 

 

Scenario 2

Create 100 Single Calendar Events (see batch functions first, then execute)

  • Go to the Create -> Bulk Create tab in the control panel
  • Set the Asset Type to Single Calendar Event
  • Make sure Extra attribute on create is checked and set start_dateto any date e.g. 2014-05-06
    (NOTE: This is intended for required fields. Some asset types, like Calendar events require a start date when created. You may set further attributes easily once they are created.)
  • Set the Parent Asset Id. This is where they will be created.
  • Leave Create Type on Normal
  • Set Number of assets to create to 100
  • Set the Asset name to Number %index% Single Calendar Event

Make sure Test is checked and click Create assets

This will show you the batch functions without executing them

If you're happy, go back to the Create -> Bulk Create tab, uncheck Test and press Create assets to start creating

If Load after created is checked, they will be loaded into the table as soon as they are finished. Then you can manipulate them further.

 

 

Scenario 3

Create a site structure from a CSV

  • Go to the Create -> Structure Create tab in the control panel
  • Set the Parent Asset Id. This is where the site structure will be created.
  • Paste your CSV into the Structure CSV textarea (there's a placeholder CSV in there that is valid)
    The format is ,My Page Name(asset_type_code),
    (NOTE: preceding and succeeding whitespace is removed, empty lines are ignored)
  • Use the Available Asset Types dropdown as a reference for the correct asset type codes to use in the brackets

Click Check CSV

If there are any errors with the CSV, they will be in the Reporting tab

If the CSV is fine, details will be shown of how many of each asset type will be created

When you are happy with your CSV, click Create Structure

Your site structure will be created and the new assets will be loaded into the table.

 

 

Scenario 4

Execute a function on 100 assets: Set metadata

  • Load some assets into the table (see scenario 1)
  • Go to the Execute -> Function tab in the control panel
  • Set the function to Set Metadata.
  • Set the field_id to your metadata field e.g. Note (125)
  • Set the field_val to Hello, I'm asset %index% in the table, my name is %name% and I am a %type_code%
    • There's a dropdown for the available keyword replacements to use in the value field
      Check the Test checkbox and click Execute to see what kind of results you'll get from the different keywords
  • Uncheck Test and click Execute to set the metadata

 

 

Scenario 5

Execute a function on 100 assets: Update link, type 1 to type 2

  • Load some assets into the table (see scenario 1)
  • Go to the Execute -> Function tab in the control panel
  • Set the function to Update Link.
  • Set the parent_id
  • Click the set checkbox and set the existing_link_type to SQ_LINK_TYPE_1
  • Click the set checkbox and set the link_type to SQ_LINK_TYPE_2
  • Click Execute to update the links

 

 

Scenario 6

Execute a function on 100 assets: Set attribute 'Allow Unrestricted Access' to No

  • Load some assets into the table (see scenario 1)
  • Go to the Execute -> Function tab in the control panel
  • Set the function to Set Attribute.
  • Set the attr_name to allow_unrestricted
    • NOTE: this must be the id of the field (e.g. short_name,title,allow_unrestricted)
    • You can find available attributes' ids from the Available keywords dropdown just below or the Display fields select in the top left
  • Set the attr_val to 0
  • Click Execute to set the attributes

 

 

Scenario 7

Execute a one off function on the server: e.g. Get Asset Types

  • Go to the Execute -> Function tab in the control panel
  • Set the function to Get Asset Types
    • No assets are used in this function so this can be executed without loading any assets
  • Click Execute to get the asset types

 

 

Scenario 8

Get a CSV (id,name,shortname,1 metadata field) of some assets, ordered by: status(asc),created(desc),id(asc)

  • Load some assets into the table (see scenario 1)
  • In the Display fields select box in the top left, select Name (name), Short name (short_name) and your metadata field (e.g. Note (#125))
    • Id is displayed by default
    • Select as many fields as you want, by all means you can build a huge CSV if you need to
      The further columns will disppear off the page but your CSV will have all the values
  • In the Sorting dropdowns on the right:
    • In the first dropdown select Status, uncheck the descending checkbox
    • In the second dropdown select Created, check the descending checkbox
    • In the first dropdown select Id, uncheck the descending checkbox
  • Go to the Reporting tab and click Get CSV
    • This will produce a CSV of whatever is displaying in the table above, including formatted dates so you may want to disable that

 

 

Scenario 9

Get a CSV of all the assets' ids in the table, separated by a semicolon

  • Load some assets into the table (see scenario 1)
  • Go to the Reporting tab and click Get CSV
  • In the divider text field, type in the character ;
    • You can also type in tab, which will put in a tab character (good for pasting into excel!)
  • Select Horizontal from the radio buttons
  • The ids are available along the top of the CSV

 

 

Scenario 10

Update names and metadata for 100 assets with a CSV

  • Prepare a CSV of some assets with the fields that you want to change (see scenario 8)
  • Go to the Execute -> Bulk CSV Change tab in the control panel
  • Paste your CSV into the CSV textarea
    • Make sure your columns are using the field ids e.g. id,name,short_name,125
    • Make sure you have an Id column on the far left
  • Click Check for changes, this will:
    • Check the CSV for too many/missing values,
    • Load all the assets (using the ids) into the table
    • Compare values of the CSV with those currently in Matrix
    • Load the changes into the table on the right
  • As you make changes, feel free to click Check for changes over and over to update the changes table on the right
    • Reload will query the assets from Matrix again
      Uncheck this if you want to compare the CSV with the assets already loaded (much faster!)
  • If you are happy with the changes to be made, click Make changes

 

 

Scenario 11

Change the format of the dates in the table (e.g. 2014-11-27 14:34:58)

The date formatting makes use of moment.js. You can see a list of the available format characters here.

Improvement to be made:It will try to process every value in the table so it will interpret some values, like Standard Page 5 as 2001-05-01 00:00:00. This will be resolved soon!

  • Load some assets into the table (see scenario 1)
  • Under Dates check Format dates
    • The default format uses the fromNow() function (e.g. Two days ago)
  • Check Manual format to override this with your own format
  • Enter Do \o\f MMM 'YY \a\t HH:mm
